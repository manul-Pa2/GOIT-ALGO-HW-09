_____Жадібний(Greedy) алгоритм vs Dynamic Programming_______

Номінали монет: [50, 25, 10, 5, 2, 1]

*----- Реалізовано ------
- "find_coins_greedy(amount)" — жадібний алгоритм
- "find_min_coins(amount)" — динамічне програмування

*----- Приклад -----
Для "amount = 113":
- Greedy: {50: 2, 10: 1, 2: 1, 1: 1}
- DP: {1: 1, 2: 1, 10: 1, 50: 2}

*---- Складність -----
- Greedy:
  - Час: O(k), де "k" — кількість номіналів (що вказує на тип "O(1)").
  - Пам’ять: O(1).
- Dynamic Programming:
  - Час: O(amount * k).
  - Пам’ять: O(amount).


*----- Висновок -------

Жадібний алгоритм працює значно швидше на великих сумах, бо робить лише один прохід по списку номіналів... - що ми і вчили в теоретичній частині.
Алгоритм динамічного програмування повільніший і потребує більше пам’яті, але гарантує мінімальну кількість монет для довільних систем номіналів (коли жадібний може давати не оптимальний результат).
